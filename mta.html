Multitier architecture

Multi-tier architecture, also known as n-tier architecture, is a design pattern in software engineering where an application is divided into multiple logical layers, or “tiers,” each responsible for specific aspects of the application‘s functionality.

Client tier

The Presentation Tier handles all user interactions but doesn‘t include any business logic. Its focused purely on displaying data and gathering inputs. This separation allows changes to the interface without affecting the core application.

Application Layer

The Application Tier is responsible for processing data from the Presentation Tier and applying business logic. It acts as the middle layer, handling operations, workflows, and communicating with the Data Tier.

Data Tier

The Data Tier is the foundation of the architecture, responsible for managing and storing data. This tier usually consists of databases, but it can also include file storage systems, external APIs, or cloud-based services.
